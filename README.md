# ScholarAI — AI-Powered Study Assistant

> Advanced Topics in Machine Learning · HTML Course Project

ScholarAI transforms your lecture notes and slides into a complete study package — summary, interactive quiz, audio podcast, and explainer video — all powered by a deep ML pipeline running entirely on CPU.

## What It Does

Upload a PDF or PowerPoint and ScholarAI generates:

- **Summary** — Abstractive summary using Google Pegasus (pegasus-xsum), with sentences selected via BGE embeddings compressed through a custom Autoencoder
- **Quiz** — 10 multiple-choice questions with 4 options, correct answer, and reasoning, generated by a T5 question-generation transformer (valhalla/t5-base-qg-hl)
- **Audio** — Text-to-speech narration of the summary using gTTS
- **Video** — Slide-style explainer video using MoviePy + PIL, with duration dynamically calculated from content length

All outputs are saved to your Firebase account so you never have to regenerate.

## ML Pipeline

```
PDF / PPTX
    ↓
Text Extraction (PyMuPDF / python-pptx)
    ↓
Sentence Tokenization (NLTK)
    ↓
BGE Embeddings (BAAI/bge-small-en-v1.5)  ← 384-dim
    ↓
Semantic Autoencoder (PyTorch)            ← 384 → 128-dim latent space
    ↓
Centroid-based Sentence Selection
    ↓
┌─────────────────────────────────────────────────┐
│  Pegasus (google/pegasus-xsum)  → Summary        │
│  T5 QG  (valhalla/t5-base-qg-hl) → MCQ Quiz     │
│  gTTS                            → Audio MP3     │
│  MoviePy + PIL                   → Video MP4     │
└─────────────────────────────────────────────────┘
    ↓
Firebase Firestore (persist results per user)
```

### Autoencoder Architecture

The autoencoder is the core Advanced ML component. It compresses high-dimensional BGE sentence embeddings into a compact 128-dimensional latent space, removing redundancy and surfacing core semantic signals before feeding into Pegasus.

```
Encoder                          Decoder
────────                         ────────
Input:  384-dim (BGE)            Latent: 128-dim
Dense:  512  + ReLU + BN         Dense:  256  + ReLU + BN
Dense:  256  + ReLU + BN         Dense:  512  + ReLU + BN
Dense:  128  + Tanh  ← latent →  Output: 384-dim
```

Training objective: MSE reconstruction loss with Adam optimizer and gradient clipping.
Evaluation metrics: Reconstruction MSE, cosine similarity, compression ratio (3:1).

## Local Development

### Prerequisites
- Python 3.11+
- pip

### Backend

```bash
# Install dependencies
pip install -r requirements.txt

# Download spaCy model
python -m spacy download en_core_web_sm

# Start the API server
uvicorn backend.app:app --host 0.0.0.0 --port 8000 --reload
```

API docs available at: `http://localhost:8000/docs`

### Frontend

```bash
# Serve frontend locally
python -m http.server 3000 --directory frontend
```

Open: `http://localhost:3000/login.html`

> Make sure `frontend/firebase-config.js` exists with your Firebase credentials.
> Copy `firebase-config.example.js` → `firebase-config.js` and fill in your values.

## Deployment

### Automated (GitHub Actions)

Every push to `main` automatically:
1. Deploys the frontend to **GitHub Pages**
2. Pushes the backend to **Hugging Face Spaces**

Required GitHub Secrets:

| Secret | Value |
|---|---|
| `HF_TOKEN` | Hugging Face write token |
| `HF_SPACE_ID` | `your-hf-username/ScholarAI-backend` |
| `HF_SPACE_URL` | `https://your-hf-username-scholarai-backend.hf.space` |
| `FIREBASE_CONFIG_JSON` | Your Firebase config as a single-line JSON object |

### Manual

**Backend (Hugging Face Spaces):**
1. Create a new Space → SDK: Docker → Hardware: CPU Basic (free)
2. Push this repo to the Space's git remote

**Frontend (GitHub Pages):**
1. Repo Settings → Pages → Branch: `gh-pages`
2. Run the GitHub Actions workflow once to populate `gh-pages`

## Security

- Uploaded files are **AES-256 encrypted** (Fernet) before being written to disk
- Files are **auto-deleted** after 1 hour via a background cleanup thread
- Firestore rules restrict each user to their own data only
- Firebase API key is injected at deploy time via GitHub Secrets — never stored in the repo

## Course Context

This project was built for the **Advanced Topics in Machine Learning (HTML)** course. The key academic contributions are:

1. **Autoencoder for semantic compression** — A custom PyTorch autoencoder reduces BGE embedding dimensionality from 384 to 128, demonstrating unsupervised representation learning
2. **Transformer-based summarization** — Pegasus (gap-sentence generation pre-training) outperforms BART on abstractive document summarization
3. **Seq2seq question generation** — T5 fine-tuned on question generation replaces rule-based heuristics with a proper ML approach
4. **End-to-end pipeline** — All components are integrated into a single FastAPI service with a production-ready frontend
